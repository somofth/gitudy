### 🎮 Git-Vis 퀴즈 시나리오 (Total 6 Levels)

#### **Level 1. 프로젝트 시작 (git init)**

> "모든 일의 시작은 창고 정리부터!"

* **상황:** 아직 아무런 시스템이 없는 상태. 화면의 네 가지 공간이 전부 회색으로 비활성화되어 있음.
* **지문:** "새로운 프로젝트를 시작하려고 해! 이 폴더를 Git이 관리하도록 설정해줄래?"
* **시각적 단서:** [작업 공간]에 불이 꺼져 있음.
* **선택지:** `git init` | `git start` | `git open` | `git help`
* **정답:** **`git init`**
* **성공 피드백:**
    * (애니메이션) 💡 **탁!** 하고 작업 공간에 불이 들어오며 활성화됨.
    * (메시지) "좋아! 이제 이 공간은 Git 감시 하에 있어. 관리자가 배치됐어!"

---

#### **Level 2. 이삿짐 담기 (git add)**

> "바닥에 널브러진 서류, 일단 박스에 담자."

* **상황:** [작업 공간] 바닥에 빨간색 서류(파일)가 있음.
* **지문:** "코드를 다 짰어! (수정 완료). 이걸 기록으로 남기려면, 우선 어떻게 해야 좋지..?"
* **시각적 단서:** [작업 공간]에서 [임시 저장]으로 가는 길이 필요함.
* **선택지:** `git commit` | `git push` | `git add` | `git move`
* **정답:** **`git add`**
* **오답 유도(함정):** `git commit`을 누르면 -> "📦 박스에 담지도 않았는데 포장할 순 없어! 먼저 담아줘."
* **성공 피드백:**
    * (애니메이션) 서류가 슝 날아가서 [임시 저장] 박스에 쏙 들어감.
    * (메시지) "나이스 샷! 이제 포장할 물건을 다 골랐어."

---

#### **Level 3. 포장 및 기록 (git commit)**

> "박스를 테이프로 밀봉하고 견출지 붙이기."

* **상황:** [임시 저장]에 서류가 담긴 파일이 놓여 있음.
* **지문:** "박스에 물건을 다 담았어. 어떻게 해야 좋지..?"
* **시각적 단서:** [임시 저장]의 박스가 대기 중. [로컬 저장소]는 비어 있음.
* **선택지:** `git save` | `git commit` | `git status` | `git add`
* **정답:** **`git commit`**
* **성공 피드백:**
    * (메시지) "기록 완료! 언제든 이 상태로 시간을 되돌릴 수 있게 됐어."

---

#### **Level 4. 물류센터로 발송 (git push)**

> "내 컴퓨터가 고장 나면 어떡해?"

* **상황:** [로컬 저장소]에 파란색 박스가 하나 쌓여 있음. [원격 저장소]는 텅 비어 있음.
* **지문:** "이제 팀원 민수랑도 협업할 수 있게 준비해야겠어."
* **선택지:** `git upload` | `git pull` | `git push` | `git send`
* **정답:** **`git push`**
* **오답 유도(함정):** `git pull`을 누르면 -> "🚚 엥? 물건을 보내야지 가져오면 안 되지!"
* **성공 피드백:**
    * (애니메이션) 파일이 [원격 저장소]로 이동함.
    * (메시지) "발송 성공! 이제 민수도, 아니 누구도 이 코드를 볼 수 있어."

---

#### **Level 5. 친구의 작업물 가져오기 (git pull)**

> "친구가 작업을 끝냈대. 내 방으로 가져오자."

* **상황:** (갑자기 알림음 띠링!) [원격 저장소]에 내가 모르는 '노란색 박스'가 새로 생김. [작업 공간]에는 그 박스가 없음.
* **지문:** "어? 민수가 새 기능을 만들어서 올렸대! 최신 내용을 **내 방으로 가져와서** 합쳐야 해."
* **선택지:** `git clone` | `git push` | `git fetch` | `git pull`
* **정답:** **`git pull`**
* **참고:** `git clone`도 헷갈릴 수 있는데, 여기선 "이미 작업 중인 프로젝트의 변경 사항 가져오기"이므로 `pull`이 맞음. 오답 힌트로 "clone은 아예 처음 복제할 때 쓰는 거야!"라고 알려주면 됨.
* **성공 피드백:**
    * 노란색 박스가 [원격 저장소]에서 [작업 공간]으로 이동함.
    * (메시지) "동기화 완료! 이제 친구가 만든 코드 위에서 계속 작업할 수 있어."
